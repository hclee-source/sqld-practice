Oracle 문법 자동 변환

Oracle 문법을 그대로 입력하고 실행하면 자동으로 SQLite 문법으로 변환됩니다.
변환된 항목은 실행 결과 상단에 표시됩니다.

Oracle 입력자동 변환 결과
NVL(COMM, 0)IFNULL(COMM, 0)
MOD(SAL, 1000)(SAL % 1000)
SYSDATEdate('now')
WHERE ROWNUM <= 5LIMIT 5
MINUSEXCEPT
TO_CHAR(dt, 'YYYY')STRFTIME('%Y', dt)
TO_DATE(str, fmt)date(str)
ADD_MONTHS(dt, 3)date(dt, '+3 month')

※ CONNECT BY, DECODE(), NVL2() 등 복잡한 Oracle 전용 문법은 자동 변환이 어렵습니다.
우측 예제 패널에서 SQLite 대체 문법을 확인하세요.

6 · Oracle 문법 자동 변환

이 환경은 SQLite 기반이지만 Oracle 문법을 그대로 입력해도 자동 변환됩니다. 변환된 항목은 결과창 상단에 표시됩니다.

Oracle 입력자동 변환 결과
NVL(COMM, 0)IFNULL(COMM, 0)
SYSDATEdate('now')
MOD(SAL, 1000)(SAL % 1000)
WHERE ROWNUM <= 5LIMIT 5
MINUSEXCEPT
TO_CHAR(dt, 'YYYY')STRFTIME('%Y', dt)
ADD_MONTHS(dt, 3)date(dt, '+3 month')
⚠ 자동 변환 불가 (SQLite 미지원): CONNECT BY / START WITH · DECODE() · PIVOT / UNPIVOT · LISTAGG() · ROWID · DUAL
→ 우측 예제 패널 [계층형 질의], [PIVOT · Top N] 에서 대체 구문을 확인하세요.

8 · 자주 하는 실수 & 주의사항
  • WHERE 없이 UPDATE / DELETE 실행 시 전체 행이 변경됩니다 — 실수했다면 브라우저를 새로고침(F5)하면 DB가 원래 상태로 복구됩니다
  • GROUP BY에 없는 컬럼을 SELECT에 단독으로 쓰면 오류 발생
  • AVG() · COUNT(컬럼)은 NULL 행 자동 제외 — COUNT(*)와 결과가 다를 수 있음
  • Oracle의 날짜 직접 비교는 SQLite에서 문자열 비교로 동작 — 날짜 형식을 통일해서 작성 필요
  • 브라우저 탭을 닫아도 진행도와 에디터 내용은 자동 저장됨 (같은 브라우저 기준)
